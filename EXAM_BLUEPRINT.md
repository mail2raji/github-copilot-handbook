# Exam Blueprints — what each GitHub exam tests, and where this book covers it

This handbook prepares you for **five** Microsoft / GitHub certifications. Each section below maps the official exam blueprint to the phase/chapter of this book.

> Exam outlines change. Always verify the current outline at [examregistration.github.com](https://examregistration.github.com/) before scheduling. The mappings below are accurate at the time of writing (2026).

---

## 1. GitHub Foundations

**Audience:** Anyone using GitHub for the first time.
**Format:** ~75 multiple-choice questions, 120 minutes, online proctored.
**Domains (approximate weights):**

| Domain | Weight | Book coverage |
|---|---|---|
| Introduction to Git | 13% | Phase 1 |
| Working with GitHub repositories | 19% | Phase 2 |
| Collaboration features (issues, PRs, discussions) | 22% | Phase 2 + Phase 3 |
| Modern development with Actions, Packages, Pages, Codespaces, Copilot | 21% | Phase 4 + Phase 5 |
| Project management with Projects | 8% | Phase 2 §2.9 |
| Privacy, security and administration | 9% | Phase 8 |
| Benefits of the GitHub community | 8% | Phase 2 §2.10 |

**Mock exam:** Phase 9 → Mock A.

## How to use this blueprint

1. Read all 11 phases once, end-to-end.
2. Take **Mock A** (Foundations). Score < 80%? Re-read flagged chapters.
3. Repeat for Mocks B, C, D, E in any order.
4. Schedule the exam **only** after you score ≥ 85% on the corresponding mock.

> Tip: Exams allow re-takes. But the *real* prize is being able to **build** what the exam tests — Phase 10 capstones get you there.
